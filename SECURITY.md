# 安全说明

Lan Control Hub 面向个人局域网多设备协作，不是公网远控服务。请只在自己信任的家庭、工作室或实验网络里使用。

## 安全边界

- 加入家庭网络需要加入密钥。
- 控制消息使用家庭密钥派生的 AES-GCM 密钥加密。
- 每台设备有独立 Ed25519 身份密钥，控制消息会签名校验。
- 控制消息会检查时间戳和消息 ID，防止旧控制包被重复投递。
- 新安装默认需要手动信任设备。
- 撤销信任后，该设备不能继续控制、浏览文件库或上传文件。
- 本机 Local API 只监听 `127.0.0.1`，并使用本机随机 token。

## 建议用法

- 不要把加入密钥发给不信任的人。
- 在多人 Wi-Fi 或办公室网络中保持“自动信任新设备”关闭。
- 不需要文件中枢时关闭“文件库共享”。
- 对只需要观察的设备开启只读模式。
- 如果加入密钥泄露，重新创建家庭网络，并让自己的设备重新加入。

## 已知限制

- 目前没有云账号体系，信任关系保存在每台设备本地。
- 目前文件传输仍是单文件消息，默认限制 100 MB。
- Windows Release 暂未做代码签名，macOS Release 暂未做 Apple Developer ID 签名和 notarization，首次打开可能需要在系统安全设置里确认。
