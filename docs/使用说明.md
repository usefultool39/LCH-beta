# Lan Control Hub 使用说明

这份文档面向普通使用者，说明每台电脑第一次怎么加入、加入后怎么聊天、传文件、远控、执行命令，以及遇到问题怎么处理。

## 1. 基本概念

### 家庭网络

家庭网络就是你自己的局域网控制圈。加入同一个家庭网络的电脑会互相发现，但新设备默认还需要手动信任后才可控。

### 加入密钥

加入密钥是把新电脑加入家庭网络用的。

已经加入的电脑，比如 L002、L012，在设置页里看到的加入密钥，是给下一台新电脑复制用的。它不是登录密码，也不是让当前电脑再次输入的东西。

### 设备名称、别名、房间

- 设备名称：电脑自己的系统名，例如 L002。
- 别名：你给它起的好记名字，例如“客厅主机”“书房 Mac”。
- 房间：用于分组，例如“客厅”“卧室”“工作室”。

建议每台电脑加入后立刻设置别名和房间，这样以后远控时不容易点错。

## 2. 第一台电脑创建网络

1. 打开 Lan Control Hub。
2. 在首次页面选择“我是第一台，创建网络”。
3. 创建后进入主界面。
4. 打开“设置”。
5. 在“添加新设备”里复制加入密钥。

这台电脑会成为家庭网络中的第一台设备。它不是服务器，其他设备不依赖它一直开着；每台设备都是对等节点。

## 3. 新电脑加入网络

1. 把 Windows 版放到 Windows 电脑上，macOS 版放到 Mac 上。
2. 打开 Lan Control Hub。
3. 选择“我已有加入密钥”。
4. 粘贴已加入电脑复制出来的加入密钥。
5. 点击加入。
6. 加入成功后，在两台电脑的“设置”里分别确认待信任设备并点击“信任”。
7. 回到任意一台已加入电脑的“多设备工作台”，应该能看到这台新电脑。

如果没看到，先等 5 到 10 秒。设备发现依赖局域网广播，偶尔需要一两个广播周期。

## 4. 多设备工作台

主界面就是“多设备工作台”。

左侧：

- 搜索设备。
- 按房间/分组查看设备。
- 勾选多个设备作为批量命令目标。

中间：

- 当前选中设备。
- 远程控制主按钮。
- 看屏、终端、批量命令入口。
- 所有设备卡片。

右侧：

- 修改别名和房间。
- 收藏设备。
- 设置只读模式。
- 查看最近任务。
- 查看当前远控会话。

## 5. 远程控制

1. 在工作台选择目标设备。
2. 点击“远程控制”。
3. 会打开一个独立远控窗口。
4. 看到远端画面后，把鼠标移到画面上即可控制远端电脑。
5. 右侧工具区可发送快捷键、读取/写入远端剪贴板。
6. 不想继续控制时，关闭窗口或点击停止。

### 鼠标位置不准怎么办

目前已经做了高 DPI 坐标修正和鼠标移动事件合并。如果仍然偏移：

- 先确认远端 Windows 显示缩放是否过高，比如 150%、175%。
- 尽量让远控窗口完整显示远端画面，不要缩得太小。
- 如果是中文输入，优先用“写入剪贴板 + Ctrl+V”。

后续可以继续做“点击四角校准”的精确校准模式。

## 6. 看屏和截图

看屏适合只观察，不一定要控制。

1. 选择设备。
2. 点击“截屏/看屏”。
3. 如果实时流没出来，点击“刷新截图”。

截图模式也可以配合 CLI 智能体使用：先截图观察，再点击/输入。

### WebRTC 连接配置

实时看屏和远程控制使用 WebRTC。默认 ICE 服务器为空，优先走局域网直连。

如果设备在 Tailscale、ZeroTier、WireGuard 或复杂 NAT 环境下能互相发现但实时流不稳定，可以在“设置 -> 网络 -> WebRTC 连接”添加 `stun:` 或 `turn:` 服务器。TURN 用户名和凭据只保存在本机，不会广播给其他设备。

一般情况下保持“直连优先”。只有明确希望所有实时流都走 TURN 时，再切到“仅中继”。

## 7. 远程命令

在工作台的命令输入框里输入命令，例如：

```bash
hostname
whoami
pwd
```

可以选择单台或多台设备执行。结果会按设备保存到任务日志里。

Windows 默认使用 PowerShell。macOS 默认使用当前用户 shell。不会自动提权，不会绕过 UAC/sudo。

## 8. 交互终端

1. 选择设备。
2. 点击“终端”。
3. 输入命令并按 Enter。

新版会优先使用 PTY + xterm，支持 ANSI 颜色、光标控制、交互程序、Tab 和窗口尺寸变化。如果远端环境无法加载 PTY，会自动降级到原来的基础终端。终端适合连续操作，单次命令适合用“远程命令”。

## 9. 聊天

1. 左侧选择设备。
2. 点击聊天。
3. 输入文字发送。

聊天是轻量对话，不依赖外网。当前支持：

- 本地搜索消息文本、文件名和引用内容。
- 点击消息气泡里的回复按钮，发送带引用摘要的新消息。
- 点击 reaction 按钮给消息做轻量反馈。
- 文本消息支持安全 Markdown 子集：`**加粗**`、`` `代码` `` 和 fenced code block。

如果对方还是旧版本，普通文本仍能收到；回复和 reaction 会自动降级或只在新版一侧显示。

当前版本会在聊天页左侧显示直接会话和群组会话。点击“新建群组”后选择成员即可创建群组会话；选中群组后可以点击“管理”修改标题和成员。群组 metadata 会通过 `conversation.upsert` 尽量同步给在线可信成员；群组文本和聊天里的小文件也会携带会话 metadata，并通过现有加密通道逐个发送给成员。群组共享目录流式大文件、已读回执和更完整的多人协作能力会放到后续版本。

## 10. 文件和共享目录

### 发送单个文件

1. 进入聊天或文件功能。
2. 选择文件发送。
3. 对方会收到文件记录。

### 共享目录

1. 在“文件”页选择本机共享目录。
2. 其他设备可以浏览这个目录。
3. 远端可以下载共享目录里的文件。

建议共享目录只放需要共享的内容，不要直接共享整个系统盘。

## 11. Windows 防火墙

如果设备在线，但控制、命令、截图超时，通常是目标电脑 Windows 防火墙阻止了 Lan Control Hub。

在目标电脑上处理：

1. 打开 Lan Control Hub。
2. 进入“设置”。
3. 找到“Windows 防火墙”。
4. 点击“修复防火墙”。
5. Windows 弹出权限确认时选择允许。

也可以在目标电脑运行：

```bash
lch firewall status --json
lch firewall repair
```

修复后，`blockRules` 应该是 `0`。

## 12. macOS 权限

macOS 首次看屏或远控时，需要在系统设置里允许：

- 屏幕录制。
- 辅助功能。

如果没有授权，App 应能提示权限问题。授权后建议重启 App。

## 13. 给智能体使用

智能体不需要点 GUI，可以直接用 CLI：

```bash
lch devices --json
lch observe --device 远端测试机 --out screen.png --json
lch click --device 远端测试机 --x 0.5 --y 0.5
lch type --device 远端测试机 "hello"
lch hotkey --device 远端测试机 ctrl+s
lch run --device 远端测试机 "hostname" --timeout-ms 30000 --json
```

推荐智能体的基本循环：

1. `devices` 找设备。
2. `observe` 看屏幕。
3. `click/type/hotkey/clipboard` 操作 GUI。
4. `run` 执行命令行任务。
5. `tasks` 读取结果。
