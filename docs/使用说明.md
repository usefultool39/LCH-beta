# Lan Control Hub 使用说明

这份文档面向普通使用者，说明每台电脑第一次怎么加入、加入后怎么聊天、传文件、远控、执行命令，以及遇到问题怎么处理。

如果要检查界面是否足够简单、设置页和 API 入口应该怎么分层，请看 [界面与操作设计](界面与操作设计.md)。

如果要按当前阶段做完整测试，请先看 [阶段总结 v0.15.1](阶段总结-v0.15.1.md)。

## v0.13 房间化加入流程

首次使用是“房间 + 网络入口”模型：

1. 如果同一局域网里已经有一台电脑创建了房间，新电脑打开后先点击“扫描”，在附近入口列表里选择要加入的房间。
2. 从房主电脑或已加入电脑的设置页复制“房间密码 / 加入密钥”，粘贴到新电脑。
3. 加入后，设置页会显示当前房间名、房间代码和本机代号。
4. 如果加入错房间，可以在设置页“网络”页签点击“退出当前房间”，回到创建/加入页面后重新加入。
5. 如果扫描不到任何入口，说明当前局域网里还没有已打开并已加入房间的设备；这台电脑可以先创建新房间，或跨网时直接粘贴房间密钥。

房间扫描只用于发现当前局域网里的入口；真正加入仍需要房间密码/加入密钥，设备控制仍需要信任。Tailscale 的 `100.x.x.x` 不是另一个房间，而是同一房间的跨网入口。

## 1. 基本概念

### 家庭网络

家庭网络/房间就是同一套加入密钥和信任关系。设备可以通过局域网 `192.x.x.x`、Tailscale `100.x.x.x` 或手动入口互相连接；网络入口会变，房间不会跟着变。

### 加入密钥

加入密钥是把新电脑加入家庭网络用的。

已经加入的电脑，比如 L002、L012，在设置页里看到的加入密钥，是给下一台新电脑复制用的。它不是登录密码，也不是让当前电脑再次输入的东西。

### 设备名称、别名、房间

- 设备名称：电脑自己的系统名，例如 L002。
- 别名：你给它起的好记名字，例如“客厅主机”“书房 Mac”。
- 房间：用于分组，例如“客厅”“卧室”“工作室”。

建议每台电脑加入后立刻设置别名和房间，这样以后远控时不容易点错。

## 2. 第一台电脑创建网络

1. 打开 Lan Control Hub。
2. 在首次页面选择“我是第一台，创建网络”。
3. 创建后进入主界面。
4. 打开“设置”。
5. 在“添加新设备”里复制加入密钥。

这台电脑会成为家庭网络中的第一台设备。它不是服务器，其他设备不依赖它一直开着；每台设备都是对等节点。

## 3. 新电脑加入网络

1. 把 Windows 版放到 Windows 电脑上，macOS 版放到 Mac 上。
2. 打开 Lan Control Hub。
3. 选择“我已有加入密钥”。
4. 粘贴已加入电脑复制出来的加入密钥。
5. 点击加入。
6. 加入成功后，在两台电脑的“设置”里分别确认待信任设备并点击“信任”。
7. 回到任意一台已加入电脑的“多设备工作台”，应该能看到这台新电脑。

如果没看到，先等 5 到 10 秒。设备发现依赖局域网广播，偶尔需要一两个广播周期。

## 4. 多设备工作台

主界面就是“多设备工作台”。

左侧：

- 搜索设备。
- 按房间/分组查看设备。
- 勾选多个设备作为批量命令目标。

中间：

- 当前选中设备。
- 远程控制主按钮。
- 看屏、终端、批量命令入口。
- 所有设备卡片。

右侧：

- 修改别名和房间。
- 收藏设备。
- 设置只读模式。
- 查看最近任务。
- 查看当前远控会话。

## 5. 远程控制

1. 在工作台选择目标设备。
2. 点击“远程控制”。
3. 会打开一个独立远控窗口。
4. 看到远端画面后，把鼠标移到画面上即可控制远端电脑。
5. 右侧工具区可发送快捷键、读取/写入远端剪贴板。
6. 不想继续控制时，关闭窗口或点击停止。

### 鼠标位置不准怎么办

目前已经做了高 DPI 坐标修正和鼠标移动事件合并。如果仍然偏移：

- 先确认远端 Windows 显示缩放是否过高，比如 150%、175%。
- 尽量让远控窗口完整显示远端画面，不要缩得太小。
- 如果是中文输入，优先用“写入剪贴板 + Ctrl+V”。

后续可以继续做“点击四角校准”的精确校准模式。

## 6. 看屏和截图

看屏适合只观察，不一定要控制。

1. 选择设备。
2. 点击“截屏/看屏”。
3. 如果实时流没出来，点击“刷新截图”。

截图模式也可以配合 CLI 智能体使用：先截图观察，再点击/输入。

### WebRTC 连接配置

实时看屏和远程控制使用 WebRTC。默认 ICE 服务器为空，优先走局域网直连。

如果设备在 Tailscale、ZeroTier、WireGuard 或复杂 NAT 环境下能互相发现但实时流不稳定，可以在“设置 -> 网络 -> WebRTC 连接”添加 `stun:` 或 `turn:` 服务器。TURN 用户名和凭据只保存在本机，不会广播给其他设备。

一般情况下保持“直连优先”。只有明确希望所有实时流都走 TURN 时，再切到“仅中继”。

## 7. 远程命令

在工作台的命令输入框里输入命令，例如：

```bash
hostname
whoami
pwd
```

可以选择单台或多台设备执行。结果会按设备保存到任务日志里。

Windows 默认使用 PowerShell。macOS 默认使用当前用户 shell。不会自动提权，不会绕过 UAC/sudo。

## 8. 交互终端

1. 选择设备。
2. 点击“终端”。
3. 输入命令并按 Enter。

新版会优先使用 PTY + xterm，支持 ANSI 颜色、光标控制、交互程序、Tab 和窗口尺寸变化。如果远端环境无法加载 PTY，会自动降级到原来的基础终端。终端适合连续操作，单次命令适合用“远程命令”。

## 9. 聊天

1. 左侧选择设备。
2. 点击聊天。
3. 输入文字发送。

聊天是轻量对话，不依赖外网。当前支持：

- 本地搜索消息文本、文件名和引用内容。
- 点击消息气泡里的回复按钮，发送带引用摘要的新消息。
- 点击 reaction 按钮给消息做轻量反馈。
- 文本消息支持安全 Markdown 子集：`**加粗**`、`` `代码` `` 和 fenced code block。

如果对方还是旧版本，普通文本仍能收到；回复和 reaction 会自动降级或只在新版一侧显示。

当前版本会在聊天页左侧显示直接会话和群组会话。点击“新建群组”后选择成员即可创建群组会话；选中群组后可以点击“管理”修改标题和成员。群组 metadata 会通过 `conversation.upsert` 尽量同步给在线可信成员；群组文本和聊天里的小文件也会携带会话 metadata，并通过现有加密通道逐个发送给成员。群组共享目录流式大文件、已读回执和更完整的多人协作能力会放到后续版本。

## 10. 文件和共享目录

### 发送单个文件

1. 进入聊天或文件功能。
2. 选择文件发送。
3. 对方会收到文件记录。

### 共享目录

1. 在“文件”页打开“本机文件库”。
2. 默认情况下，其他可信设备可以浏览桌面、下载、文档、图片、视频、音乐和你选择的自选目录。
3. 远端可以下载共享目录里的文件，也可以上传到当前可写目录。

### 完整磁盘访问

如果你要让另一台可信电脑看到本机所有可见磁盘，在被访问的电脑上打开：

1. 进入“文件中枢”。
2. 找到“本机文件库”。
3. 勾选“允许可信设备浏览本机所有可见磁盘”。
4. 远端文件根目录会出现 `C:`、`D:` 等盘符；CLI 可以使用 `lch file list --device 远端测试机 "C:/"`。

这个开关默认关闭。它只影响 LCH 文件中枢，不会绕过 Windows/macOS 权限；需要管理员权限的目录，仍然需要在目标电脑用管理员身份启动 App。

## 11. Windows 防火墙

如果设备在线，但控制、命令、截图超时，通常是目标电脑 Windows 防火墙阻止了 Lan Control Hub。

在目标电脑上处理：

1. 打开 Lan Control Hub。
2. 进入“设置”。
3. 找到“Windows 防火墙”。
4. 点击“修复防火墙”。
5. Windows 弹出权限确认时选择允许。

也可以在目标电脑运行：

```bash
lch firewall status --json
lch firewall repair
```

修复后，`blockRules` 应该是 `0`。

## 12. macOS 权限

macOS 首次看屏或远控时，需要在系统设置里允许：

- 屏幕录制。
- 辅助功能。

如果没有授权，App 应能提示权限问题。授权后建议重启 App。

## 13. 手机控制台

手机和电脑在同一局域网时，打开：

```text
http://电脑IP:46882/mobile/
```

如果手机和一台电脑都连接 Tailscale、ZeroTier 或 WireGuard，就把电脑 IP 换成虚拟专网 IP。手机名称随便填；密码使用同一个房间的房间密钥。

手机端页签：

1. 总览：看房间和在线数量，执行快捷动作。
2. 控制：选择一台设备，和该设备上已配置的 CLI 智能体进行聊天式控制。
3. 设备：查看每台设备状态，包括在线、信任、只读和 CLI 是否配置。
4. 任务：看最近输出。

开启 Agent Gateway 后，“控制”页会列出可选设备。当前只有已配置 Claude Code / MiniMax-M3 的设备会显示“CLI 可用”；未配置的设备会显示“CLI 未配置”，不能发送到智能体。语音只会填入控制输入框，不会自动执行；确认文字后再点“发送”。

## 14. 给智能体使用

智能体不需要点 GUI，可以直接用 CLI：

```bash
lch devices --json
lch observe --device 远端测试机 --out screen.png --json
lch click --device 远端测试机 --x 0.5 --y 0.5
lch type --device 远端测试机 "hello"
lch hotkey --device 远端测试机 ctrl+s
lch run --device 远端测试机 "hostname" --timeout-ms 30000 --json
```

推荐智能体的基本循环：

1. `devices` 找设备。
2. `observe` 看屏幕。
3. `click/type/hotkey/clipboard` 操作 GUI。
4. `run` 执行命令行任务。
5. `tasks` 读取结果。
