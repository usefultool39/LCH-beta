# 公开发布检查清单

这份清单用于把仓库切换为公开源码仓库，以及发布新的 GitHub Release 前做最后确认。

## 公开仓库前

1. 确认 `.gitignore` 覆盖 `node_modules/`、`dist/`、`release/`、运行日志、截图视频、`state.json` 和 `local-api.json`。
2. 运行公开扫描：

```bash
npm run public:scan
```

3. 确认仓库中没有本机状态文件、家庭加入密钥、设备私钥、Local API token、release 安装包或测试截图。
4. 确认 `LICENSE` 存在，README 说明源码许可证和 release 下载方式。
5. 确认不需要公开历史中的旧 release 二进制或本机配置；如果扫描失败，先清理再公开。

## 发布 v0.14.0

```bash
npm ci
npm run check
npm run public:scan
git status
git add .
git commit -m "prepare v0.14.0 release"
git push
git tag v0.14.0
git push origin v0.14.0
```

GitHub Actions 会校验 `package.json`、`APP_VERSION`、tag 和 README 中的 artifact 名称是否一致。

## Release 附件

`v0.14.0` 应包含：

```text
Lan-Control-Hub-0.14.0-win-x64-portable.exe
Lan-Control-Hub-0.14.0-win-x64-setup.exe
Lan-Control-Hub-0.14.0-mac-x64.zip
Lan-Control-Hub-0.14.0-mac-arm64.zip
SHA256SUMS.txt
```

## 手工验收

1. Windows 设备和 macOS 设备分别下载对应 release。
2. 设备 A 创建网络并复制加入密钥。
3. 设备 B 使用加入密钥加入。
4. 两台设备都能看到对方处于待信任状态。
5. 两边互相信任后，聊天、文件浏览/下载/上传、截图/看屏、远控输入都可用。
6. 撤销信任后，对方不能继续控制、浏览文件库或上传文件。
7. 两边重启 App 后，身份和信任关系仍保留。

## 未签名说明

`v0.14.0` 暂不要求付费签名证书。Windows SmartScreen 和 macOS Gatekeeper 可能提示来源确认；这是未签名个人工具的预期行为，不代表 release workflow 失败。
