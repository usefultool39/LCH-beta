# 安全与权限

这份文档说明 Lan Control Hub 在局域网内如何识别设备、限制访问，以及使用时建议怎么配置。

## 核心原则

局域网不等于完全可信。Lan Control Hub 的目标是让自己的多台电脑互相协作，同时避免同一个网络里的陌生设备直接看到文件或控制电脑。

当前安全边界由三层组成：

1. 家庭网络加入密钥。
2. 每台设备自己的身份密钥。
3. 本机对其他设备的信任列表和只读设置。

## 加入密钥

第一台电脑创建家庭网络后会生成加入密钥。新设备必须输入这个密钥，才能算作同一个家庭网络。

加入密钥用于：

- 生成家庭网络 ID。
- 派生控制消息加密密钥。
- 让同一家庭网络内的设备互相发现并通信。
- 控制消息会携带消息 ID 和时间戳；接收端会拒绝过旧或重复的控制消息。

不要把加入密钥发给不信任的人。如果密钥泄露，建议重新创建家庭网络，再让自己的设备重新加入。

## 设备身份

每台设备首次运行时会生成一对身份密钥。设备之间的控制消息会携带签名，接收方会验证签名和设备公钥。

如果某台设备的身份密钥变化，通常意味着：

- App 数据目录被清空。
- 这台电脑重新安装后生成了新身份。
- 有其他设备冒用了旧设备名称。

遇到身份变化时，不要只看设备名，应该在设置页检查设备 ID 和公钥指纹，再决定是否重新信任。

## 自动信任和手动信任

新安装默认关闭自动信任。这样即使有人在同一个局域网里拿到了 App，也不能只靠 Wi-Fi 自动获得访问权限。

推荐用法：

1. 保持“自动信任新设备”关闭。
2. 新设备出现后会显示为“待信任”。
3. 确认设备名、IP 和公钥指纹后，手动点“信任”。

撤销信任后，该设备即使还在同一局域网，也不能继续发送控制消息、浏览文件库或上传文件。

只有在确定这个 Wi-Fi 里全是自己的设备时，才建议开启自动信任。

## 只读模式

对某台设备开启只读后，它不能对本机执行写入或控制类动作。

只读会阻止：

- 远程命令。
- 交互终端。
- 远程控制输入。
- 剪贴板写入。
- 上传文件到本机文件库。
- 发送文件到本机默认接收目录。

只读仍允许读取类操作，例如看屏、截图、浏览文件库和下载文件。需要完全阻止文件读取时，请关闭本机文件库共享或撤销设备信任。

## 文件库共享范围

文件中枢默认不会暴露整块磁盘。默认共享的文件库根目录包括：

- 桌面
- 下载
- 文档
- 图片
- 视频
- 音乐
- 自选共享目录

如果你确实要把目标电脑当成自己的远程工作站，可以在目标电脑的“文件中枢 / 本机文件库”里开启“允许可信设备浏览本机所有可见磁盘”。开启后，远端根目录会额外显示 `C:`、`D:` 这类盘符以及系统能看到的外接盘。

远端传入的路径会被规范化并限制在这些根目录内。未开启完整磁盘访问时，绝对路径、盘符路径和 `..` 跳出目录的路径都会被拒绝；开启后，`C:/Users/...` 这类路径只允许留在对应盘符根目录内，不能通过 `..` 跳到其它根。Windows/macOS 系统权限仍然生效，需要管理员权限的系统目录必须在目标电脑用管理员身份启动 App。

## 建议配置

个人家庭网络：

- 推荐保持手动信任；如果网络里只有自己的设备，可以开启自动信任。
- 给常用设备设置别名和房间。
- 对不常用设备开启只读。

办公室或多人网络：

- 保持自动信任关闭。
- 只信任自己的设备。
- 不用文件中枢时关闭文件库共享。
- 定期检查已信任设备列表，撤销不用的设备。

## 已知限制

- 加入密钥泄露后，同一家庭网络的加密边界会变弱，需要重建家庭网络。
- 聊天里的直接文件发送仍保留 100 MB 上限；文件中枢下载和上传已走 token + HTTP 流式传输，单文件上限为 20 GB，并支持媒体 Range 预览。
- 当前没有账号体系；信任关系保存在每台设备本地。
