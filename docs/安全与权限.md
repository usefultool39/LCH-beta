# 安全与权限

这份文档说明 Lan Control Hub 在局域网内如何识别设备、限制访问，以及使用时建议怎么配置。

## 核心原则

局域网不等于完全可信。Lan Control Hub 的目标是让自己的多台电脑互相协作，同时避免同一个网络里的陌生设备直接看到文件或控制电脑。

当前安全边界由三层组成：

1. 家庭网络加入密钥。
2. 每台设备自己的身份密钥。
3. 本机对其他设备的信任列表和只读设置。

## 加入密钥

第一台电脑创建家庭网络后会生成加入密钥。新设备必须输入这个密钥，才能算作同一个家庭网络。

加入密钥用于：

- 生成家庭网络 ID。
- 派生控制消息加密密钥。
- 让同一家庭网络内的设备互相发现并通信。

不要把加入密钥发给不信任的人。如果密钥泄露，建议重新创建家庭网络，再让自己的设备重新加入。

## 设备身份

每台设备首次运行时会生成一对身份密钥。设备之间的控制消息会携带签名，接收方会验证签名和设备公钥。

如果某台设备的身份密钥变化，通常意味着：

- App 数据目录被清空。
- 这台电脑重新安装后生成了新身份。
- 有其他设备冒用了旧设备名称。

遇到身份变化时，不要只看设备名，应该在设置页检查设备 ID 和公钥指纹，再决定是否重新信任。

## 自动信任和手动信任

默认保持旧体验：同一个家庭网络里的新设备会自动加入信任列表，适合只有自己设备的家庭网络。

更安全的用法：

1. 在设置页关闭“自动信任新设备”。
2. 新设备出现后会显示为“待信任”。
3. 确认设备名、IP 和公钥指纹后，手动点“信任”。

撤销信任后，该设备即使还在同一局域网，也不能继续发送控制消息、浏览文件库或上传文件。

## 只读模式

对某台设备开启只读后，它不能对本机执行写入或控制类动作。

只读会阻止：

- 远程命令。
- 交互终端。
- 远程控制输入。
- 剪贴板写入。
- 上传文件到本机文件库。
- 发送文件到本机默认接收目录。

只读仍允许读取类操作，例如看屏、截图、浏览文件库和下载文件。需要完全阻止文件读取时，请关闭本机文件库共享或撤销设备信任。

## 文件库共享范围

文件中枢不会暴露整块磁盘。默认共享的文件库根目录包括：

- 桌面
- 下载
- 文档
- 图片
- 视频
- 音乐
- 自选共享目录

远端传入的路径会被规范化并限制在这些根目录内。绝对路径、盘符路径和 `..` 跳出目录的路径都会被拒绝。

## 建议配置

个人家庭网络：

- 可以开启自动信任。
- 给常用设备设置别名和房间。
- 对不常用设备开启只读。

办公室或多人网络：

- 关闭自动信任。
- 只信任自己的设备。
- 不用文件中枢时关闭文件库共享。
- 定期检查已信任设备列表，撤销不用的设备。

## 已知限制

- 加入密钥泄露后，同一家庭网络的加密边界会变弱，需要重建家庭网络。
- 当前文件传输以单文件 base64 消息传输为主，默认限制 100 MB，超大文件建议后续改为分块传输。
- 当前没有账号体系；信任关系保存在每台设备本地。
