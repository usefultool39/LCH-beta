# 安装配对与发布

这份文档说明普通使用者如何从 GitHub Releases 下载 App，以及两台设备如何互相信任。

## 从 GitHub Releases 下载

打开仓库的 Releases 页面：

```text
https://github.com/usefultool39/LCH-beta/releases
```

Windows 下载：

```text
Lan-Control-Hub-版本号-win-x64-setup.exe
```

或者便携版：

```text
Lan-Control-Hub-版本号-win-x64-portable.exe
```

macOS 下载：

```text
Lan-Control-Hub-版本号-mac-x64.zip
Lan-Control-Hub-版本号-mac-arm64.zip
```

Intel Mac 选 `x64`，Apple Silicon/M1/M2/M3/M4 选 `arm64`。

每个 Release 还会提供 `SHA256SUMS.txt`。下载后可以用系统自带工具核对安装包哈希，确认文件没有传输损坏。

当前 `v0.14.0` 暂未做 Windows 代码签名、Apple Developer ID 签名和 notarization：

- Windows SmartScreen 可能提示“未知发布者”，需要确认这是自己从项目 Release 页面下载的文件。
- macOS 首次打开可能需要在“系统设置 -> 隐私与安全性”里允许打开。

## 第一台设备创建家庭网络

1. 在第一台电脑安装并打开 Lan Control Hub。
2. 点击“我是第一台，创建网络”。
3. 进入“设置”。
4. 点击“显示密钥”或“复制加入密钥”。
5. 把加入密钥发到自己的第二台电脑上。

加入密钥只用于自己的设备加入，不要发给陌生人。

## 第二台设备加入

1. 在第二台电脑安装并打开 Lan Control Hub。
2. 点击“我已有加入密钥”。
3. 粘贴第一台电脑复制的加入密钥。
4. 点击加入。
5. 等待 5 到 10 秒，让两台设备通过局域网广播发现彼此。

## 双向信任

为了避免“同一个 Wi-Fi 里的人装了 App 就能看你文件”，新设备默认不会直接获得权限。

在第一台电脑上：

1. 打开“设置”。
2. 在“待信任设备”里找到第二台电脑。
3. 核对设备名、IP、公钥指纹。
4. 点击“信任”。

在第二台电脑上也做一遍：

1. 打开“设置”。
2. 在“待信任设备”里找到第一台电脑。
3. 核对后点击“信任”。

两边都信任后，设备工作台会显示在线，文件、聊天、命令、终端、看屏和远控才会正常可用。

## 文件中枢怎么用

1. 选择远端设备。
2. 打开“文件”页。
3. 进入远端文件库，例如桌面、下载、文档、图片。
4. 点击文件会下载到本机。
5. 把本机文件拖到远端目录区域，会上传到远端当前目录。

如果不想让别人读取本机文件，在本机“文件”页关闭“文件库共享”。

## 发布新版 Release

源码更新并通过检查后，创建版本 tag：

```bash
npm run check
npm run public:scan
git status
git add .
git commit -m "prepare v0.14.0 release"
git push
git tag v0.14.0
git push origin v0.14.0
```

推送 tag 后，GitHub Actions 会自动：

1. 在 Windows 上打包 `setup.exe` 和 `portable.exe`。
2. 分别在 macOS Intel runner 和 Apple Silicon runner 上打包 `x64.zip` 和 `arm64.zip`。
3. 创建 GitHub Release。
4. 生成 `SHA256SUMS.txt`。
5. 从 `CHANGELOG.md` 读取 release notes。
6. 把安装包和校验文件上传到 Release 附件。

如果 Actions 失败，先打开仓库的 Actions 页面查看具体失败日志。
