## 已知 Bug

### alias / displayName 字段在 API 输出层被截断 — Fixed in b377b99

**状态**：已修复。设备列表现在会把别名、系统名、设备代号和连接入口分开展示；`lch devices --json` 也会输出独立的 `display`、`name`、`alias`、`current` 和 `routes` 字段。

如果仍看到旧乱码，通常是本机还在运行旧客户端。退出托盘里的 Lan Control Hub，安装最新版后重新启动即可。

### desktop App 安装版不附带 `lch` CLI 到 PATH — Fixed in v0.18.0

**状态**：已修复。`package.json` 的 `build.asarUnpack` 现在包含
`scripts/lch.js` 和 `scripts/lch.cmd`，它们会落到
`resources/app.asar.unpacked/scripts/`，可被 Node 直接 require。

App 内增加 "lch 命令行 (CLI)" 卡片（设置 → 系统）：
- 点击「添加 lch 到命令」会把 `lch.exe` 写入
  `HKCU\Software\Microsoft\Windows\CurrentVersion\App Paths\lch.exe`，
  指向 `resources\app.asar.unpacked\scripts\lch.cmd`。新打开的
  PowerShell / cmd 即可直接跑 `lch devices`、`lch run`、`lch file get`。
- 不再需要把 `lch.js` 路径写进 `$PROFILE`。

macOS / Linux 暂未实现，但 LCH 仓库根目录的
`scripts/lch.cmd` / `scripts/lch.js` 在源码场景下仍然可用。

# 方式 3：用 Local API（port 从 local-api.json 读）
$cfg = Get-Content "$env:APPDATA\lan-control-hub\local-api.json" | ConvertFrom-Json
curl "http://127.0.0.1:$($cfg.port)/api/devices" -H "Authorization: Bearer $($cfg.token)"
```

**修复建议**：
在 `electron-builder` 的 NSIS 配置里加一段"添加 lch 到 PATH"的可选步骤，
或者在打包时把 `scripts/lch.js` 复制到 `app.asar.unpacked/cli/lch.js` 并附带
一个 `lch.cmd` wrapper：
```cmd
@echo off
set "LCH_LOCAL_API_CONFIG=%APPDATA%\lan-control-hub\local-api.json"
node "%~dp0resources\app.asar.unpacked\cli\lch.js" %*
```

## 用户容易踩到的坑

### PowerShell `$_` 被吞

PowerShell 在调用外部命令时会把字符串里的 `$_` 转换为 `_`，所以 `lch run` 传入的
命令里**别用 `$_` 表达式**。

```powershell
# 不行 — `$_` 被吞，PowerShell 在 L002 那侧解析失败
Get-PSDrive | Select-Object @{N='GB';E={[math]::Round($_.Used/1GB,1)}}

# 用属性名（推荐）
Get-Volume | Format-Table DriveLetter,FileSystemLabel,Size,SizeRemaining -AutoSize

# 或用 $PSItem 替代 $_
Get-PSDrive | Select-Object @{N='GB';E={[math]::Round($PSItem.Used/1GB,1)}}
```

详细报错通常是 `MissingEndParenthesisInMethodCall`，因为 `$_` 变 `_` 后
PowerShell 解析不完整。

---

### local-api.json 端口可能不是默认 46882

如果 46882 被占用，App 启动时会自增端口（46882 + N，N ≤ 30）。
调 Local API 时要用 actual port：

```powershell
$cfg = Get-Content "$env:APPDATA\lan-control-hub\local-api.json" | ConvertFrom-Json
# {"app":"Lan Control Hub","port":46883,"token":"..."}

curl "http://127.0.0.1:$($cfg.port)/api/devices" -H "Authorization: Bearer $($cfg.token)"
```

---

### 跨网时不会自动发现设备

UDP 局域网广播只在同一 IP 子网有效。跨网（不同 WiFi、不同 VLAN、4G 热点）
必须手动 `peer add`：

```powershell
lch peer add 100.x.x.x
# 或在 App 设置 → 网络 → 跨网入口 / Tailscale 页面添加
```

详见 `外网访问推荐配置.md`。

---

### `lch run --device X` 不支持自动 mesh 路由

`lch run` 只会直连 `peer.address`（即 trustedDevices 里记录的那个地址），
不通就报错。**如果目标电脑没装 Tailscale，必须走 Agent Gateway 中转**：

1. 中间机器开启 Agent Gateway（设置 → 系统 → 高级工具）
2. 通过 `mobile-api/commands/run`（不是 local-api）调用
3. 中间机器代理执行命令

详见 `移动端控制台.md` 第 88-100 行。

TypeError: Cannot set properties of undefined (setting 'lastCheckedAt')
