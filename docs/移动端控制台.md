# 移动端控制台

`mobile/` 是 Lan Control Hub 的手机 Web/PWA 控制台。它不是单独原生 App，而是由任意一台已经加入房间的电脑通过 Web 端口 `46882` 提供。

## 访问地址

同局域网时，手机打开：

```text
http://电脑局域网IP:46882/mobile/
```

例如当前这台电脑可以使用：

```text
http://192.168.2.93:46882/mobile/
```

如果手机和其中一台电脑都连接 Tailscale、ZeroTier 或 WireGuard，则把局域网 IP 换成那台电脑的虚拟专网 IP：

```text
http://电脑TailscaleIP:46882/mobile/
```

不要把 `46882` 直接做公网端口映射。

## 名称、密码和 IP 的区别

- 手机名称：随便填，例如“我的手机”“iPhone”，只用于日志和会话显示。
- 房间密钥：同一个 LCH 房间共用一个密钥，不是每台电脑一套密码。
- IP 地址：代表手机访问哪一台电脑作为入口。基础模式只操作这台入口电脑；开启 Agent Gateway 后，才能通过它把命令转发到同房间内其它在线可信设备。

## 手机端页面

手机端默认是基础控制面板，分成三个底部页签：

- 总览：看房间、网关、在线设备数量，执行网关本机常用快捷动作。
- 设备：查看设备状态；基础模式下只有网关本机可作为操作目标。
- 任务：查看最近命令输出。

在桌面端“设置 / 系统 / 高级工具”开启 Agent Gateway 后，手机端会额外显示“智能体”和“命令”页：

- 智能体：固定连接当前网关电脑上的 Claude Code，并以 `MiniMax-M3` 模型启动。手机可以用输入框或语音把内容发送给这个会话，页面会持续刷新输出。
- 命令：面向一次性 shell 命令和批量设备命令，可选择目标设备、使用预设或语音填入。

界面按手机优先设计，按钮和文字都应该在窄屏上可读，不再使用桌面端压缩布局。

## 快捷动作

快捷动作适合常用、低输入成本的操作。基础模式下只作用于网关本机；开启 Agent Gateway 后才允许转发到其它在线可信设备：

- 连接测试：执行 `hostname; whoami`。
- 查看网络：读取目标设备网络配置。
- 打开下载目录：在目标电脑打开下载目录。
- 锁定屏幕：锁定目标电脑屏幕，执行前需要确认。

## 手机智能体

手机端“智能体”页属于 Agent Gateway 高级模式，默认不显示。开启后，页面提供一条固定路径：

```text
手机输入 / 语音
  -> 网关电脑的移动端 API
  -> 启动 claude --model MiniMax-M3
  -> 手机页面轮询显示输出
```

这条路径只连接网关电脑上的 Claude Code，不提供任意 CLI 下拉选择。这样可以先把“手机操作智能体”做通，同时避免把移动端扩展成一个过宽的通用终端入口。

使用方式：

1. 在桌面端开启 Agent Gateway。
2. 手机上打开 `http://电脑IP:46882/mobile/` 并登录。
3. 进入“智能体”页，点击“启动”。
4. 在输入框输入内容，或点击“语音输入”后确认发送。
5. 输出区域会随页面刷新更新；点击“停止”会关闭当前手机 token 对应的 Agent 会话。

当前固定命令：

```bash
claude --model MiniMax-M3
```

如果这台电脑的 Claude Code 没有配置 MiniMax-M3，启动后会在输出区显示 CLI 自身错误。移动端不会保存 API key，也不会替用户配置模型。

## 命令

手机端“命令”页属于 Agent Gateway 高级模式，默认不显示。开启后支持三种执行范围：

- 网关本机：命令只在当前提供手机网页的电脑上执行。适合调用这台电脑已安装的 CLI 或智能体。
- 选中设备：命令通过 LCH 转发到你在“设备”页勾选的电脑。
- 全部在线：命令在网关本机和所有在线可信远端设备上执行。

这条链路用于一次性命令：

```text
手机输入命令 -> 网关电脑执行 CLI/智能体 -> 网关电脑通过 LCH 控制多台设备
```

例如网关电脑上如果安装了 `codex`、`claude`、`opencode`、`gemini` 等 CLI，就可以在手机端选择“网关本机”，输入这些 CLI 支持的非交互命令。需要和 Claude Code 长时间来回对话时，优先使用“智能体”页。

常用预设：

- 检查设备状态：`hostname; whoami`
- 查看网关 CLI：`codex --version; claude --version; opencode --version; gemini --version`
- 全部主机名：`hostname`

命令执行前手机端会弹出确认。不要粘贴不可信脚本，不要在手机端随意运行删除、格式化、清空目录等高风险命令。

## 语音输入

手机端提供“开始语音 / 语音填入”：

- 如果语音匹配到快捷动作，例如“查看网络”“锁屏”，会先提示确认。
- 如果语音像命令，例如“运行 hostname”“在全部设备上执行 whoami”，会填入命令框并切换到命令页。
- 真正执行命令前仍然需要手动确认。

浏览器支持情况取决于手机系统和浏览器。部分浏览器只在 HTTPS 或特定环境支持 Web Speech；如果按钮不可用，可以点命令输入框，用手机键盘自带的麦克风输入。

## 外网 / Tailscale 场景

推荐方案是“只连一台电脑，当 LCH 网关”：

```text
手机 --Tailscale--> 网关电脑 --局域网/LCH控制协议--> 其它电脑
```

手机只需要访问网关电脑的 `46882/mobile/`。网关电脑已经在局域网内，所以它可以通过现有 LCH 控制通道转发快捷动作和命令到其它在线可信设备。

这不等于手机自动进入整个物理局域网；它是应用层网关。实时看屏和远控跨网关转发需要后续单独增强。

## API 形态

手机页面登录后会拿到临时 mobile token。快捷指令或自动化工具后续可以复用 token 调用 API。Agent Gateway 关闭时，命令接口会被拒绝，快捷动作只会在网关本机执行。

快捷动作：

```text
POST http://电脑IP:46882/mobile-api/actions/run
Authorization: Bearer <mobile-token>
Content-Type: application/json

{
  "actionId": "lock-screen",
  "peerIds": ["目标设备ID"]
}
```

命令：

```text
POST http://电脑IP:46882/mobile-api/commands/run
Authorization: Bearer <mobile-token>
Content-Type: application/json

{
  "mode": "selected",
  "peerIds": ["目标设备ID"],
  "command": "hostname",
  "confirm": true
}
```

`mode` 可以是：

- `gateway`：只在网关本机执行。
- `selected`：在指定设备执行。
- `all`：在全部在线可信设备执行。

智能体：

```text
POST http://电脑IP:46882/mobile-api/agent/start
Authorization: Bearer <mobile-token>

POST http://电脑IP:46882/mobile-api/agent/input
Authorization: Bearer <mobile-token>
Content-Type: application/json

{
  "text": "帮我查看当前项目状态"
}

GET http://电脑IP:46882/mobile-api/agent/state
Authorization: Bearer <mobile-token>

POST http://电脑IP:46882/mobile-api/agent/stop
Authorization: Bearer <mobile-token>
```

## 安全原则

- 不做公网端口暴露，跨网络优先用 Tailscale、ZeroTier 或 WireGuard。
- 手机登录用房间密钥换临时 mobile token。
- mobile token 保存在网关电脑内存里，应用重启后失效。
- Agent Gateway 默认关闭；预设命令、任意命令和跨设备执行必须在桌面端手动开启。
- 高危动作和任意命令都必须确认。
- 命令长度有限制，执行记录会进入任务列表和审计日志。
- 手机智能体固定为 Claude Code / MiniMax-M3，不提供任意 CLI 选择。
- 远程终端、完整远控、文件删除等更高危能力后续要做单独权限开关。
