# 移动端控制台

`mobile/` 是 Lan Control Hub 的手机 Web/PWA 版本。它不是单独原生 App，而是由任意一台已加入房间的电脑通过 Web 端口 `46882` 提供。

## 同局域网使用

1. 确认手机和电脑在同一个 Wi-Fi / 局域网。
2. 在电脑端确认 Lan Control Hub 已加入房间，并且防火墙允许 `46882`。
3. 手机浏览器打开：

```text
http://电脑局域网IP:46882/mobile/
```

例如：

```text
http://192.168.2.93:46882/mobile/
```

4. 输入手机名称和房间密钥登录。
5. 登录后可以查看设备、选择目标设备、执行白名单快捷动作、查看最近任务。

当前手机 session 只保存在提供网关的电脑内存里；Lan Control Hub 重启后，手机需要重新登录。

## 当前功能边界

手机端第一版只开放安全白名单动作：

- 连接测试：执行 `hostname; whoami`。
- 查看网络：读取目标设备网络配置。
- 打开下载目录：在目标电脑上打开下载目录。
- 锁定屏幕：锁定目标电脑屏幕。

手机端不会直接开放任意 shell。以后接入语音或大模型时，也应该先把自然语言解析成结构化意图，再映射到这些白名单动作。

## 语音命令

手机页面已经预留浏览器 Web Speech 入口。支持的浏览器上，可以点击“语音识别”，识别结果会先匹配快捷动作，再弹出确认。

建议后续按这个顺序增强：

1. 浏览器 Web Speech：快速验证手机语音入口。
2. OpenAI 或其它语音识别 API：提升普通话和自然语言识别效果。
3. 本地离线语音模型：适合对隐私和离线可用性要求更高的场景。

无论使用哪种语音识别，执行层都应该继续使用白名单动作，不要把语音文本直接当作远程终端命令。

## iOS 快捷指令和 Android 自动化

手机控制台底层是 HTTP API，后续可以给 iOS 快捷指令、Android Tasker / MacroDroid 调用。

推荐形态：

```text
POST http://电脑IP:46882/mobile-api/actions/run
Authorization: Bearer <mobile-token>
Content-Type: application/json

{
  "actionId": "lock-screen",
  "peerIds": ["目标设备ID"]
}
```

快捷指令不应该保存房间密钥；应该先由手机页面登录拿到 mobile token，再让快捷指令使用这个 token。

## 外网 / Tailscale 场景

手机离开家里的 Wi-Fi 后，不建议把 `46882` 暴露到公网。推荐继续走 Tailscale、ZeroTier 或 WireGuard。

### 方案 A：每台电脑都安装 Tailscale

这是最直接的方式。手机和每台电脑都在同一个 tailnet 里，手机打开：

```text
http://某台电脑的TailscaleIP:46882/mobile/
```

优点是简单清晰；缺点是每台电脑都要装 Tailscale。

### 方案 B：只连一台电脑，当 LCH 网关

你的设想可以成立，但不是“手机自动进入整个物理局域网”。真实结构是：

```text
手机 --Tailscale--> 网关电脑 --局域网/LCH控制协议--> 其它电脑
```

手机只需要访问网关电脑的 `46882/mobile/`。网关电脑已经在局域网里，所以它可以通过现有 Lan Control Hub 控制通道转发动作到其它已信任设备。

这个方案适合手机控制台、快捷动作、命令、文件和后续 Agent。实时看屏/远控如果要跨这层网关，后续需要专门做 WebRTC 转发或截图轮询；第一版不把它作为目标。

### 方案 C：Tailscale Subnet Router

也可以把一台电脑配置成 Tailscale subnet router，把家里局域网网段发布到 tailnet。这样手机理论上能访问局域网内其它设备的 IP。

这个方案更接近网络层桥接，但配置更复杂，需要在 Tailscale 管理端审批路由、处理防火墙和局域网网段冲突。对 Lan Control Hub 来说，优先做方案 B 的应用层网关更可控。

## 安全原则

- 不要公网端口映射。
- 手机登录使用房间密钥换临时 mobile token。
- mobile token 可撤销，应用重启后失效。
- 高危动作需要确认。
- 第一版不开放任意 shell。
- 后续远程终端、远控输入、文件删除等能力要单独做权限开关。
