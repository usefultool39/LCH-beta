# 跨网 5 分钟配置

适用场景：两台电脑在不同物理网络下（比如笔记本 4G 热点 + 台式机家里 WiFi），
希望互相通过 Lan Control Hub 控制。

完整背景参考 `外网访问推荐配置.md`，本指南是它的精简 checklist 版。

---

## 步骤

### 1. 两台都装 Tailscale（1 分钟）

```powershell
# 在两台机器上分别安装
winget install Tailscale.Tailscale
# 或从 https://tailscale.com/download/windows 下载安装
```

装好后登录**同一个 Tailscale 账号**（或被邀请进同一 tailnet）。

### 2. 验证 tailnet 互通

两台机器分别跑：

```powershell
tailscale status
# 应该看到对方节点 IP（100.x.x.x 段）

tailscale ping 100.x.x.x    # 替换成对方 Tailscale IP
# 应该几秒内返回 pong + 延迟（直连 < 100ms，DERP 中继 100-300ms）
```

### 3. 两台都装 LCH 桌面 App

从 GitHub Releases 下载跟自己匹配的安装包：

| 平台 | 文件 |
|---|---|
| Windows | `Lan-Control-Hub-0.16.0-win-x64-setup.exe` 或 `portable.exe` |
| macOS Intel | `Lan-Control-Hub-0.16.0-mac-x64.zip` |
| macOS Apple Silicon | `Lan-Control-Hub-0.16.0-mac-arm64.zip` |

启动 App，确认右下角托盘出现 LCH 图标。

### 4. 互相加入同一房间

- 在**一台** App：设置页 → 房间 → 复制"房间密钥"
- 在**另一台** App：设置页 → 房间 → 粘贴密钥加入
- **两边的密钥必须完全一致**（含大小写、空格、特殊字符）

### 5. 手动添加对方为 peer（绕过 UDP 广播限制）

跨网时 UDP 局域网广播不通，必须手动添加：

```powershell
# 在本机跑
lch peer add 100.x.x.x    # 对方 Tailscale IP
```

或者在 App 设置 → 网络 → “跨网入口 / Tailscale” 页面手动输入 IP。

预期结果：`status=online`，1-2 秒内返回成功。

### 6. 互信一次（必须双方）

两台 App 都进入 **设置 → 信任** 页签：
- 看对方的设备条目，分别点击"信任"
- 这一步**单方不算**，必须双方都点
- 撤销后即使还在同一局域网，也不能继续控制

### 7. 修复防火墙（如需要）

```powershell
lch firewall status
# 如果提示 needsAttention=true：

lch firewall repair
# 走完后再查 status
```

### 8. 验证全链路

```powershell
lch devices --json
# 应该看到对方设备，status=online，并且 current/routes 里出现 Tailscale入口:100.x.x.x:46881:current:online

lch run --device <对方名字或IP> "hostname"
# 应该能跨 tailnet 执行命令并返回结果
```

如果全部 OK，配置完成。

## 三台设备验收场景

你的常见场景可以作为跨网主链路验收：

- 笔记本使用手机热点，开启 Tailscale。
- 两台台式机在家里路由器的 `192.168.x.x` 局域网内，同时也开启 Tailscale。
- 三台设备加入同一个 LCH 房间，互相信任。
- 笔记本为两台台式机分别添加 Tailscale `100.x.x.x` 入口。

验收时在笔记本运行：

```powershell
lch devices --json
lch run --device 100.x.x.x "hostname"
lch run --device <另一台100.x.x.x> "hostname"
```

如果两台台式机都显示 `online`，`current/routes` 都有 `Tailscale入口:100.x.x.x:46881:current:online`，并且 `hostname` 都能返回，就说明跨物理网络、手动入口、同房间校验、设备信任和命令控制链路是通的。

这仍不等于覆盖了所有 LAN 行为。要完整覆盖 UI 上的两类入口，还需要在三台设备处于同一家庭路由器 `192.168.x.x` 时再验证一次：设备条目应出现 `局域网入口:192.168.x.x:46881`，并且同样能执行 `hostname`。
## 切换网络后


只要 Tailscale 在线 + LCH App 在跑，配置会自动恢复。
新网络下第一次可能要等 10-30 秒（Tailscale 重新打洞 / 切换 DERP 中继）。

## 故障排查

| 现象 | 排查 |
|---|---|
| `lch devices` 看不到对方 | `tailscale ping <对方 100.x>` 看 Tailscale 层是否通；再确认 App 的“跨网入口 / Tailscale”里已经添加对方 IP |
| `lch peer add` 超时 | 对方 App 没运行；Tailscale 离线；防火墙挡 46882 |
| `lch peer add` 报"家庭网络不匹配" | 检查房间密钥完全一致 |
| `lch run` 超时 | 对方 App 没运行 / 防火墙挡 46881 / 设备未 trust |
| 设备显示 offline 但 tailscale ping 通 | 等 10-30s，App 还在探测；或重启对方 App |
| 新网络后连接很慢 | Tailscale 在用 DERP 中继（家庭 WiFi 上行常被 CGNAT 挡） |
| `peer add` 报"不能添加本机地址" | IP 输错成本机了 |

## 进阶：网关中转模式（控制没装 Tailscale 的第三台电脑）

如果某台电脑不能/不愿装 Tailscale，但跟 L002 在同一 LAN，可以用 L002 当网关代理：

```
笔记本（Tailscale）→ L002（Agent Gateway）→ 第三台电脑（LAN）
```

**前提**：
1. L002 已经跟第三台电脑建立 LCH 信任（trustedDevices 里有它）
2. L002 开启 Agent Gateway（设置 → 系统 → 高级工具）
3. 笔记本通过 Tailscale 连 L002 的 46882，用 mobile-api 调用

```powershell
# 1. 笔记本浏览器打开 L002 的 mobile 页面换 token
#    http://100.120.218.47:46882/mobile/

# 2. 拿到 mobile token 后，用 mobile-api 转发命令
$LCH_GATEWAY = "http://100.120.218.47:46882"
$TOKEN = "<mobile-token>"
$TARGET_PEER_ID = "<第三台电脑的 device.id>"

curl -X POST "$LCH_GATEWAY/mobile-api/commands/run" `
  -H "Authorization: Bearer $TOKEN" `
  -H "Content-Type: application/json" `
  -d "{`"mode`":`"selected`",`"peerIds`":[`"$TARGET_PEER_ID`"],`"command`":`"hostname`",`"confirm`":true}"
```

详见 `移动端控制台.md` 第 88-164 行。

## 备份建议

`%APPDATA%\lan-control-hub\state.json` 是你这套配置的"灵魂"：
- 房间密钥
- 设备信任表
- manual peer 列表
- 设备偏好（别名、房间、收藏等）

建议偶尔备份（丢了等于重头配对）：

```powershell
$stamp = Get-Date -Format "yyyyMMdd-HHmmss"
Compress-Archive -Path "$env:APPDATA\lan-control-hub\state.json" `
  -DestinationPath "$env:USERPROFILE\Desktop\lch-state-$stamp.zip"
```
