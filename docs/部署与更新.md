# 部署与更新

这份文档用于把 Lan Control Hub 保存到 GitHub、重新打包，以及更新到其他电脑。

## 仓库内容

GitHub 仓库保存：

- `src/` 源码。
- `scripts/` CLI 脚本。
- `docs/` 文档。
- `build/` 图标。
- `package.json` 和 `package-lock.json`。
- TypeScript、Vite 和 Electron 配置。

GitHub 仓库不保存：

- `node_modules/`
- `dist/`
- `release/`
- 顶层测试截图和录屏。

这些内容可以从源码重新生成，或者作为 GitHub Release 附件单独上传。

## 本机开发检查

```bash
npm install
npm run typecheck
npm run build
npm test
```

如果只是修改文档，可以不打包；如果修改了主进程、渲染端或协议，建议至少运行 `typecheck` 和 `build`。

## Windows 打包

```bash
npm run package:win
```

打包完成后会生成：

```text
release/Lan-Control-Hub-0.5.1-win-x64-portable.exe
release/Lan-Control-Hub-0.5.1-win-x64-setup.exe
release/win-unpacked/Lan Control Hub.exe
```

推荐测试顺序：

1. 先运行 `release/win-unpacked/Lan Control Hub.exe`。
2. 再测试 portable exe。
3. 最后测试 installer。

## 推送到 GitHub

第一次推送：

```bash
git init
git branch -M main
git remote add origin https://github.com/usefultool39/LCH-_beta.git
git add .
git commit -m "initial lan control hub backup"
git push -u origin main
```

后续更新：

```bash
npm run check
git status
git add .
git commit -m "update lan control hub"
git push
```

如果 GitHub 要求登录，可以先运行：

```bash
gh auth login
gh auth status
```

也可以使用 Git Credential Manager 登录 HTTPS 远端。

## 更新另一台电脑

方式一：便携版。

1. 在本机运行 `npm run package:win`。
2. 把 `release/Lan-Control-Hub-0.5.1-win-x64-portable.exe` 复制到另一台电脑。
3. 关闭另一台电脑上的旧版 Lan Control Hub。
4. 运行新版 portable exe。

方式二：安装版。

1. 把 `release/Lan-Control-Hub-0.5.1-win-x64-setup.exe` 复制到另一台电脑。
2. 运行安装器覆盖安装。
3. 打开 App，确认家庭网络和信任设备仍在。

## 验收清单

更新后在两台电脑上确认：

- 两台电脑都能看到对方在线。
- `lch devices --json` 能列出远端设备。
- 文件页能看到远端文件库根目录。
- 能下载一个小文件。
- 能拖拽上传一个小文件到远端下载目录或桌面。
- 只读模式打开后，远程命令和上传会被拒绝。
- 关闭文件库共享后，远端不能浏览本机文件库。

## GitHub Releases 建议

源码用 Git 管，安装包用 GitHub Releases 管。

当前仓库已经包含 GitHub Actions 发布流程。发布新版时：

```bash
npm run check
npm run public:scan
git status
git add .
git commit -m "prepare v0.5.1 release"
git push
git tag v0.5.1
git push origin v0.5.1
```

推送 tag 后，GitHub 会自动生成：

- Windows portable exe
- Windows setup exe
- macOS x64 zip，由 `macos-15-intel` runner 构建
- macOS arm64 zip，由 `macos-15` runner 构建
- SHA256SUMS.txt 校验文件

这样仓库历史不会被大型二进制文件撑大，后续更新也更清爽。更多说明见 [安装配对与发布](安装配对与发布.md)。
