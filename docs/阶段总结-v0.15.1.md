# 阶段总结 v0.15.1

这份文档是当前阶段的 checkpoint。目标是让你可以先停在这个版本测试，而不是继续一边改一边追功能。

## 当前定位

Lan Control Hub 现在是一个以局域网为主的多设备通信和电脑控制工具。核心使用方式是：

```text
同一房间内的电脑互相信任 -> 局域网内直接通信 -> 手机或本机 CLI 通过网关电脑发起操作
```

跨网络场景不做公网端口暴露，优先使用 Tailscale、ZeroTier 或 WireGuard。手机只需要能访问一台网关电脑，再由这台电脑通过 LCH 控制其它在线可信设备。

## 当前已具备

- 房间大厅：第一台电脑创建房间，其它电脑扫描附近房间并输入同一个房间密钥加入。
- 设备发现：同房间设备会显示在线状态、设备代号、IP、版本和信任状态。
- 设备信任：新设备需要信任；支持自动信任开关和撤销信任。
- 聊天：一对一、群组文本、小文件、回复引用、搜索、Markdown/代码块和 reaction。
- 文件：文件中枢、共享目录、预览、流式下载和上传。
- 命令：单台、多台和全部设备远程命令。
- 终端：PTY/xterm 交互终端，保留基础 spawn fallback。
- 屏幕：截图、实时看屏、远程鼠标键盘和剪贴板。
- WebRTC：默认局域网直连，可配置 STUN/TURN 和仅中继。
- 手机控制台：总览、控制、设备、任务页签。默认先看设备状态；开启 Agent Gateway 后，“控制”页提供设备选择和聊天式 Claude Code / MiniMax-M3 控制入口。
- CLI/Local API：本机智能体可以通过 `lch` CLI 调用设备、命令、文件和远控能力。

## 收束版边界

本阶段收束目标是：LCH 原有局域网能力不被手机智能体改动影响，手机端先做好一个简单清晰的控制入口。

保持不变的核心能力：

- 设备发现、房间密钥、设备信任、自动/手动信任和只读设备。
- 桌面端聊天、文件中枢、远程命令、交互终端、看屏、远控、剪贴板和 WebRTC。
- Local API 和 `lch` CLI 供本机脚本或 Agent 使用。

手机端新增能力只挂在 `Agent Gateway` 高级开关后：

- Agent Gateway 关闭时，手机端仍是基础控制面板。
- Agent Gateway 开启后，手机“控制”页会列出设备，并显示“CLI 可用 / CLI 未配置”。
- 当前只有网关电脑本机的 Claude Code / MiniMax-M3 路径标记为 CLI 可用；其它电脑即使在线，也不会误接收自然语言命令。
- 手机输入和语音识别结果都进入聊天输入框，点击“发送”后才调用 CLI 智能体。

## 手机端当前用法

同局域网时，手机访问：

```text
http://电脑IP:46882/mobile/
```

当前这台电脑的局域网入口通常是：

```text
http://192.168.2.93:46882/mobile/
```

登录时：

- 名称：随便填，例如“我的手机”。
- 密码：填 LCH 房间密钥，不是 Wi-Fi 密码，也不是 Windows 密码。

手机页签：

- 总览：看房间状态、在线数量、快捷动作和语音入口。
- 控制：选择一台 CLI 可用设备，使用聊天窗口和 Claude Code / MiniMax-M3 对话。
- 设备：查看设备在线、信任、只读和 CLI 配置状态。
- 任务：查看最近输出。

## 手机到多设备的工作链路

当前收束后的手机智能体闭环：

```text
手机选择设备
  -> 只允许 CLI 可用的设备进入聊天
  -> 手机输入或语音填入
  -> Claude Code / MiniMax-M3 返回到同一个聊天窗口
```

例如当前只有这台笔记本配置了 Claude Code / MiniMax-M3，它会显示“CLI 可用”；其它两台会显示“CLI 未配置”，只能查看状态，不能把“你好”这类自然语言误当命令执行。

浏览器语音识别受手机系统和浏览器限制。如果 Web Speech 不可用，可以直接点控制输入框，用手机键盘自带麦克风输入。

## Tailscale / 外网方式

推荐结构：

```text
手机 --Tailscale--> 网关电脑 --局域网/LCH控制协议--> 其它电脑
```

手机打开：

```text
http://网关电脑TailscaleIP:46882/mobile/
```

这不是让手机自动进入整个物理局域网，而是让网关电脑作为 LCH 应用层中转。快捷动作、命令和后续 Agent 任务适合这种方式；实时看屏和远控跨网关转发仍属于后续增强。

## 当前安全边界

- 不建议公网端口映射。
- 手机登录使用房间密钥换临时 mobile token。
- mobile token 保存在网关电脑内存，应用重启后失效。
- 手机控制页不直接暴露任意 shell 命令；普通手机操作走聊天式 CLI 智能体。
- 高危动作如锁屏会再次确认。
- 命令执行记录会进入任务列表和审计日志。
- 手机端暂不提供完整交互终端；远端设备 CLI 智能体、任务队列和更细权限属于后续迭代。

## 建议测试清单

1. 三台电脑都打开 Lan Control Hub，并确认版本都是 `0.15.1`。
2. 手机打开 `http://192.168.2.93:46882/mobile/`。
3. 输入手机名称和房间密钥登录。
4. 总览页确认在线数量正确。
5. 设备页确认三台设备分别显示在线/离线、信任和 CLI 状态。
6. 控制页选择“CLI 可用”的笔记本，输入“你好”，确认回复显示在同一个聊天窗口。
7. 控制页选择“CLI 未配置”的设备，确认发送按钮不可用或提示未配置。
8. 任务页确认原有快捷动作和命令任务仍可查看。
9. 如果使用 Tailscale，改用网关电脑的 Tailscale IP 打开同一个 `/mobile/` 地址。
10. 桌面端再验证聊天、文件、终端、看屏和远控至少各打开一次。
